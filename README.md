# Т-Библиотека

# Контекст
В современном мире люди часто сталкиваются с проблемой управления своей личной библиотекой. Для решения этой проблемы предлагается создать консольное приложение под названием "T-Библиотека", которое позволит пользователям эффективно управлять своими книгами, добавлять их в библиотеку, отмечать, как прочитанные или добавлять в избранное.

# Функционал
Добавление книги в библиотеку: Пользователь сможет добавлять новые книги в библиотеку, указывая название, автора, жанр, год издания и краткое описание.

Просмотр книг из библиотеки: Пользователь сможет просматривать список всех книг в библиотеке, с возможностью сортировки по названию, автору или году издания, а также фильтрации по жанру или статусу (прочитана/не прочитана).

Добавление в избранное: Пользователь сможет добавлять книги в избранное, чтобы иметь быстрый доступ к ним.

Изменение статуса книги: Пользователь сможет изменять статус книги на "прочитана" или "не прочитана", а также удалять ее из избранного.

Избранные книги: Пользователь сможет получить список книг, основанные на его предпочтениях и истории чтения.

Удаление книги из библиотеки: Пользователь сможет удалять книги из библиотеки.

Поиск книги: Пользователь сможет искать книги, по ключевым словам, в названии, авторе или описании.

Сохранение данных в файл: Приложение будет сохранять все книги в файл, чтобы данные не терялись при закрытии программы.

Восстановление из файла: При запуске приложения оно будет загружать все книги из файла, если он существует.

# Описание реализации
Проект простой и совсем не требовательный, поэтому реализован на Python. Каждой книге при её добавлении присваивается уникальный ID на основе времени момента создания в мс; у данного подхода есть минус: при откате времени назад на системе есть риск получить две книги с одинаковым ID, поэтому лучше использовать UUID, однако тут было решено этим пренебречь в целях экономии времени, к тому же, такой (в мс) ID гораздо легче записать или даже запомнить. Данные сохраняются в файле json - с ним проще работать, хоть здесь вполне мог подойти, например, csv. Ввод команд по их условным номерам (1-8), это обычная и распространённая практика; перед каждым запросом пользователю сообщается, что он может/должен ввести.
